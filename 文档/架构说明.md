# c2rust-translate 架构说明

## 项目结构

```
src/
├── lib.rs              - 主工作流程编排
├── common_tasks.rs     - 公共任务
├── initialization.rs   - 项目初始化
├── verification.rs     - 构建验证和修复
├── builder.rs          - Cargo 构建
├── translator.rs       - C 到 Rust 翻译
├── analyzer.rs         - 代码分析
├── interaction.rs      - 用户交互
├── file_scanner.rs     - 文件扫描
├── git.rs              - Git 版本控制
└── ...
```

## 核心模块说明

### 1. common_tasks 模块（公共任务）

提供4个标准化的公共任务：

1. **execute_code_error_check**: 执行代码错误检查
2. **execute_code_warning_check**: 执行代码告警检查
3. **execute_hybrid_build_check**: 执行混合构建检查
4. **execute_translation_task**: 执行翻译任务

### 2. 主工作流程

```
1. 初始化 feature 目录
2. 执行初始验证（代码错误检查）
3. 扫描待翻译文件
4. 进入翻译循环
   4.1 执行翻译
   4.2 执行代码错误检查（带修复循环）
   4.3 执行代码告警检查（带修复循环）
5. 打印统计信息
```

## 命名规范

- `execute_*`: 执行某项任务
- `*_with_fix_loop`: 带自动修复循环的检查
- 避免使用 `gate_` 前缀（已废弃）

## 环境变量

| 变量名 | 说明 |
|--------|------|
| `C2RUST_PROCESS_WARNINGS` | 设为 `0` 或 `false` 可跳过 Phase 2（警告检测与修复），默认启用 |

## 公共任务调用示例

```rust
// 在任何需要检查的地方调用
crate::common_tasks::execute_code_error_check(feature, show_full_output)?;
crate::common_tasks::execute_code_warning_check(feature, show_full_output)?;
```
